Branches: block, br, br_if, br_table

การ branch ใน WebAssembly เกิดขึ้นเทียบกับ labeled construct เสมอ — ไม่ใช่กับ raw memory address กลุ่มคำสั่ง br จะชี้ไปที่ nesting level ไม่ใช่หมายเลขบรรทัด ดังนั้นพฤติกรรมของ br จึงขึ้นอยู่กับชนิด construct ที่มันชี้ถึงอย่างสิ้นเชิง

block และ br

หัวข้อที่มีชื่อว่า “block และ br”

block คือกลุ่มคำสั่งที่เดินไปข้างหน้าเท่านั้น เมื่อ br $label ชี้ที่ block execution จะกระโดดไปยังคำสั่งที่อยู่ หลัง end ของ block ทันที ให้คิดว่าเป็น labeled break

Annotation (result i32) บน block ระบุว่า block จะฝากค่า i32 หนึ่งตัวบน stack เมื่อออก — ไม่ว่าจะ fall through ถึง end หรือกระโดดออกด้วย br

ทิศทางของ br

หัวข้อที่มีชื่อว่า “ทิศทางของ br”

คำสั่ง br เดียวกันจะทำงานต่างกันขึ้นอยู่กับสิ่งที่มันชี้:

br_if: conditional branch

หัวข้อที่มีชื่อว่า “br_if: conditional branch”

br_if $label pop ค่า i32 condition จาก stack ถ้าไม่ใช่ศูนย์จะ branch ถ้าเป็นศูนย์จะ execute คำสั่งถัดไปต่อไป นี่คือวิธีมาตรฐานในการเขียน conditional early exit

br_table: jump table

หัวข้อที่มีชื่อว่า “br_table: jump table”

br_table รับรายการ label และค่า default มันจะ pop ค่า i32 index แล้วกระโดดไปที่ label ที่ตำแหน่ง index นั้น หรือไปที่ default ถ้า index เกินขอบเขต เทียบได้กับ switch statement ใน WAT

ทดลองรัน

หัวข้อที่มีชื่อว่า “ทดลองรัน”

🦾 br ใน block ออก; br ใน loop วนซ้ำ

เมื่อ br $label ชี้ที่ block มันจะกระโดดไปข้างหน้าถึงคำสั่งหลัง end — เป็น early exit เมื่อชี้ที่ loop มันจะกระโดดไปข้างหลังถึงจุดเริ่มต้นของ loop — เป็นการวนซ้ำ คำสั่งเดียวกัน แต่คนละทิศทาง
