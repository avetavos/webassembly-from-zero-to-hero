WebAssembly เป็น stack machine แบบมีชนิด (typed) ค่าทุกตัวบน stack มีชนิดที่ชัดเจน และฟังก์ชันทุกตัวประกาศอย่างแน่ชัดว่ามันรับและผลิตชนิดใดบ้าง แต่ต่างจากภาษาส่วนใหญ่ Wasm มีชุดของชนิดให้เลือกที่เล็กมาก

## ชนิดของค่าทั้งสี่

WebAssembly มีชนิดของค่าทั้งหมดสี่ชนิดพอดี:

| Type | Width | Kind |
|------|-------|------|
| `i32` | 32-bit | Integer |
| `i64` | 64-bit | Integer |
| `f32` | 32-bit | Float |
| `f64` | 64-bit | Float |

ในระดับ Wasm ไม่มีสตริง บูลีน อาเรย์ หรืออ็อบเจกต์ ทุกอย่างเป็นตัวเลข ตรรกะแบบบูลีนถูกแทนด้วยค่า `i32` โดยที่ `0` หมายถึง false และค่าใด ๆ ที่ไม่ใช่ศูนย์หมายถึง true

## การ push ค่าคงที่ขึ้นไปบน stack

แต่ละชนิดมีคำสั่ง `const` ที่ push ค่าลิเทอรัล:

```wasm
(module
  (func (export "answer") (result i32)
    i32.const 42))
```

สำหรับ float 64 บิต:

```wasm
(module
  (func (export "pi") (result f64)
    f64.const 3.14159265358979))
```

และสำหรับจำนวนเต็ม 64 บิต — สังเกตว่า `i64` มีความพิเศษเมื่อข้ามไปยัง JavaScript:

```wasm
(module
  (func (export "bignum") (result i64)
    i64.const 1000000000000))
```

## i64 กับ BigInt ของ JavaScript

ชนิด `number` ของ JavaScript เป็น float 64 บิต ซึ่งสามารถแทนจำนวนเต็มได้อย่างแม่นยำถึง 2^53 เท่านั้น แต่ `i64` สามารถเก็บค่าที่เกินกว่านั้นไปไกลมาก เพื่อรักษาความแม่นยำเต็มที่ WebAssembly JavaScript API จะแปลงค่าที่คืนกลับชนิด `i64` ให้เป็น `BigInt` แทนที่จะเป็น `number` นั่นหมายความว่า `typeof result === 'bigint'` — ไม่ใช่ `'number'`

คุณสามารถตรวจสอบสิ่งนี้ได้ใน runner ด้านล่าง

export const valueTypesCode = `const wat = \`(module
  (func (export "pi") (result f64)
    f64.const 3.14159265358979)
  (func (export "bignum") (result i64)
    i64.const 1000000000000))\`;
const bytes = await compileWat(wat);
const { instance } = await WebAssembly.instantiate(bytes, {});
console.log('pi() =', instance.exports.pi());
console.log('bignum() =', instance.exports.bignum());
console.log('bignum is BigInt:', typeof instance.exports.bignum() === 'bigint');`;
