WebAssembly มีชุดคำสั่งทางคณิตศาสตร์และการเปรียบเทียบที่หลากหลาย แต่ละคำสั่งมีชนิด (typed) — คุณรู้เสมอว่ากำลังทำงานกับจำนวนเต็มหรือ float และสำหรับจำนวนเต็ม การดำเนินการนั้นเป็นแบบมีเครื่องหมายหรือไม่มีเครื่องหมาย

## การคำนวณกับจำนวนเต็ม

การดำเนินการพื้นฐานสี่อย่างสำหรับ `i32` คือ `i32.add`, `i32.sub`, `i32.mul` และการหาร ทั้งหมด pop ตัวถูกดำเนินการสองตัวและ push ผลลัพธ์หนึ่งตัว:

```wasm
(module
  (func (export "add") (param $a i32) (param $b i32) (result i32)
    local.get $a
    local.get $b
    i32.add)
  (func (export "sub") (param $a i32) (param $b i32) (result i32)
    local.get $a
    local.get $b
    i32.sub)
  (func (export "mul") (param $a i32) (param $b i32) (result i32)
    local.get $a
    local.get $b
    i32.mul))
```

คำสั่งเดียวกันนี้มีอยู่สำหรับ `i64` เช่นกัน: `i64.add`, `i64.sub`, `i64.mul`

## การหารแบบมีเครื่องหมายเทียบกับไม่มีเครื่องหมาย

การหารเป็นจุดที่ชนิดจำนวนเต็มเผยให้เห็นความซับซ้อนของมัน `i32` ก็แค่ 32 บิต — ตัวบิตเองไม่ได้พกพาข้อมูลเครื่องหมายใด ๆ **คำสั่ง** เป็นตัวตัดสินว่าจะตีความบิตเหล่านั้นเป็นจำนวนเต็มแบบมีเครื่องหมายในรูป two's-complement หรือเป็นจำนวนเต็มแบบไม่มีเครื่องหมาย:

```wasm
(module
  ;; signed: -7 / 2 = -3 (rounds toward zero)
  (func (export "divSigned") (param $a i32) (param $b i32) (result i32)
    local.get $a
    local.get $b
    i32.div_s)

  ;; unsigned: the same bit pattern as -7 (0xFFFFFFF9) / 2 = 2147483644
  (func (export "divUnsigned") (param $a i32) (param $b i32) (result i32)
    local.get $a
    local.get $b
    i32.div_u))
```

ในทำนองเดียวกัน `i32.rem_s` และ `i32.rem_u` ให้เศษเหลือแบบมีเครื่องหมายและไม่มีเครื่องหมาย

## การคำนวณกับ float

คำสั่งสำหรับ float สะท้อนคำสั่งของจำนวนเต็ม แต่คำนำหน้าชนิด (type prefix) เปลี่ยนไป:

```wasm
(module
  (func (export "fadd") (param $a f64) (param $b f64) (result f64)
    local.get $a
    local.get $b
    f64.add))
```

`f64.sub`, `f64.mul`, `f64.div`, `f64.sqrt`, `f64.floor`, `f64.ceil`, `f64.min` และ `f64.max` มีให้ใช้ทั้งหมด ชุดเดียวกันนี้มีอยู่สำหรับ `f32` เช่นกัน

## คำสั่งเปรียบเทียบ

การเปรียบเทียบจะ pop ค่าสองค่าและ push ผลลัพธ์ `i32`: `1` หมายถึง true, `0` หมายถึง false ไม่มีชนิดบูลีน — WebAssembly ใช้ `i32` เป็นบูลีนเมื่อใดก็ตามที่ต้องการ

```wasm
(module
  ;; returns 1 if $a equals $b, else 0
  (func (export "eq") (param $a i32) (param $b i32) (result i32)
    local.get $a
    local.get $b
    i32.eq)

  ;; signed less-than: returns 1 if $a < $b (signed), else 0
  (func (export "ltSigned") (param $a i32) (param $b i32) (result i32)
    local.get $a
    local.get $b
    i32.lt_s))
```

ตัวแปรของการเปรียบเทียบเป็นไปตามรูปแบบ signed/unsigned เดียวกัน: `i32.lt_s` เทียบกับ `i32.lt_u`, `i32.gt_s` เทียบกับ `i32.gt_u`, `i32.le_s`, `i32.ge_u` และอื่น ๆ

## ลองรันดู

ตัวอย่างด้านล่างคอมไพล์โมดูลที่มีการลบ การหารแบบมีเครื่องหมาย และการเปรียบเทียบแบบน้อยกว่าชนิดมีเครื่องหมาย

export const arithmeticCode = `const wat = \`(module
  (func (export "sub") (param $a i32) (param $b i32) (result i32)
    local.get $a
    local.get $b
    i32.sub)
  (func (export "divS") (param $a i32) (param $b i32) (result i32)
    local.get $a
    local.get $b
    i32.div_s)
  (func (export "isLess") (param $a i32) (param $b i32) (result i32)
    local.get $a
    local.get $b
    i32.lt_s))\`;
const bytes = await compileWat(wat);
const { instance } = await WebAssembly.instantiate(bytes, {});
console.log('sub(10, 3) =', instance.exports.sub(10, 3));
console.log('divS(-7, 2) =', instance.exports.divS(-7, 2));
console.log('isLess(3, 5) =', instance.exports.isLess(3, 5), '(1 = true)');
console.log('isLess(5, 3) =', instance.exports.isLess(5, 3), '(0 = false)');`;
