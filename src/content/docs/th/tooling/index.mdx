การส่ง WebAssembly ที่มีประสิทธิภาพเริ่มต้นก่อนที่คุณจะเขียนคำสั่งแรก คุณต้องมีเครื่องมือเพื่อแปลงข้อความ `.wat` ให้เป็นไบนารี `.wasm` เพื่อตรวจสอบว่ามีอะไรอยู่ในโมดูลที่คอมไพล์แล้ว และเพื่อลดขนาดไบต์ก่อนที่จะส่งผ่านเครือข่าย โปรเจกต์สามตัวครอบคลุมความต้องการเกือบทั้งหมด: **wabt**, **Binaryen** และ **wasm-tools**

## ตระกูลเครื่องมือทั้งสาม

| ตระกูลเครื่องมือ | ดูแลโดย | จุดแข็งหลัก |
|---|---|---|
| **wabt** (WebAssembly Binary Toolkit) | WebAssembly CG | การแปลง WAT ↔ binary, การ validate, การตรวจสอบ |
| **Binaryen** | Google / Emscripten | การ optimise และ code generation, `wasm-opt` |
| **wasm-tools** | Bytecode Alliance | ชุดเครื่องมือที่เขียนด้วย Rust: parse, validate, compose, encode |

แต่ละตระกูลครอบครองขั้นตอนที่แตกต่างกันใน pipeline แผนภาพด้านล่างแสดงให้เห็นว่าไฟล์ต้นฉบับ `.wat` เดินทางผ่านเครื่องมือเหล่านี้อย่างไรจนกลายเป็นโมดูลที่ถูก deploy และ optimise แล้ว

## wabt — ชั้นการแปล

`wabt` คือ reference implementation ของ WAT text format binary หลักคือ `wat2wasm` ซึ่งแปลงไฟล์ `.wat` เป็นไบนารี `.wasm` ทิศทางย้อนกลับ — การ disassemble `.wasm` กลับเป็น WAT ที่อ่านได้ — จัดการโดย `wasm2wat` เครื่องมือเสริมสองตัวทำให้ครบชุด: `wasm-objdump` แสดง section headers และ symbol table ของไบนารีใดก็ได้ และ `wasm-validate` ตรวจสอบว่าโมดูลเป็นไปตามข้อกำหนดหรือไม่

```bash
# ติดตั้งผ่าน npm (ทำงานโดยไม่ต้องสร้าง native)
npm install -g wabt

# คอมไพล์ WAT ให้เป็น binary
wat2wasm add.wat -o add.wasm

# Disassemble กลับเป็น text
wasm2wat add.wasm -o add.wat

# ตรวจสอบ section headers
wasm-objdump -x add.wasm

# Validate โมดูล
wasm-validate add.wasm
```

## Binaryen — ตัว optimise

Binaryen คือโครงสร้างพื้นฐาน compiler สำหรับ WebAssembly ที่ทำการ optimise เครื่องมือที่ใช้กันมากที่สุดคือ `wasm-opt` ซึ่งเขียนไบนารี `.wasm` ที่มีอยู่ใหม่เพื่อผลิต output ที่เล็กลงหรือเร็วขึ้น Emscripten, AssemblyScript และ `wasm-pack` (สำหรับ Rust) ทั้งหมดเรียก `wasm-opt` โดยอัตโนมัติเป็นส่วนหนึ่งของ release builds

```bash
# ติดตั้ง
npm install -g binaryen

# Optimise เพื่อความเร็ว (ต้องการการทำงานที่รวดเร็ว)
wasm-opt -O3 input.wasm -o output.wasm

# Optimise เพื่อขนาด (ต้องการ download ที่เล็ก)
wasm-opt -Oz input.wasm -o output.wasm
```

## wasm-tools — ชุดเครื่องมือสมัยใหม่

`wasm-tools` คือชุดเครื่องมือรุ่นใหม่ของ Bytecode Alliance เขียนด้วย Rust และมีทั้งในรูปแบบ CLI และ library รองรับ WebAssembly Component Model proposals การรวมหลายโมดูลเข้าด้วยกัน การลบ custom sections และการ encode/decode ในรูปแบบ binary

```bash
# ติดตั้ง
cargo install wasm-tools

# Validate
wasm-tools validate add.wasm

# แสดง text format
wasm-tools print add.wasm

# ลบ custom sections (ลบ debug info)
wasm-tools strip add.wasm -o add.stripped.wasm
```
